# 読書ツール — APP 版本高级使用方法

经过架构重铸，目前本应用除了原本通用的“浏览器预览版”以外，正式拥有了强劲的桌面 APP 形态。本文档旨在向你指引原生环境下的终极玩法及本地金矿守护法则。

---

## 一、双重引擎：开发调试 vs. 封装发行

你可以在本地终端里通过两条核心指令，分别控制这台重型机器的不同形态。

### 1. 原生开发预览（调试模式）
> 指令：`npm run app:dev`

- **现象**：一个原生的系统级深色无边框 APP 窗口会弹出来，这就是你的专属阅读器。
- **特性**：它具有**极热更新能力**。如果你这个时候拿编辑器修改了界面的代码，保存的瞬间立刻会在这个 APP 窗口内呈现改变！特别适合对 UI 开发修修补补。 

### 2. 构建终极软件！（发行模式）
> 指令：`npm run build:mac`

- **现象**：终端开始进入漫长的编译与压缩，等几分钟读条到达 100% 后停止。
- **产物**：去你项目根目录，找到刚刚诞生不久的 `release/` 文件夹。里面那个名叫 **`読書ツール.app`** 的带图标的软件，就是完全体了。你可以把它像安装微信一样拖入 Mac底部的程序坞（Dock）当独立软件用。
- *(⚠️ 注：Windows 请使用 `npm run build:win`)*

## 二、隐形金身：私人数据去哪儿了？

为了杜绝你由于修改代码或升级功能时误删了你的辛苦阅读成果，也为了让你能在真正脱离代码环境的纯粹体验下放心记录，**APP 版本的储物柜和代码是严格隔离的！**

所有你在 APP 内阅读过书籍上传的 PDF 原件备份、**你产生的所有词汇表记录(`Vocabulary.md`)、所有语法分析档案(`Grammar.md`)以及你的专属阅读手记(`Notes.md`)** 都被安全转移到了专用的用户中心池中：

### 🎯 找回宝藏的操作办法
无论软件有无启动，随时在你的 Mac 上采取以下步骤可以直达宝库深处：
1. 切出电脑主桌面，鼠标点击页面底部最左侧像蓝色笑脸一样的 **[访达 (Finder)]**。
2. 敲击超快捷键：`Command(⌘) + Shift + G` 。此时会弹出「前往文件夹」的地址栏。
3. 一字不差地复制粘贴下面这行神奇路径，并按下回车：
```bash
~/Library/Application Support/dokusho-tool/history/
```
4. 恭喜你！一排排以你读过的书名命名的文件夹将呈现出来，你随时可以拖进 Obsidian / Typora / Logseq 一类你常用的本地笔记应用去深度消费它们了。

### 🚚 换机或系统迁移提醒
未来如果你换了一台新的 Macbook，并且想把之前的所有的读书记忆和沉淀带走：
**你毫无必要去拷代码！**
只需要把老电脑上的 `~/Library/Application Support/dokusho-tool` 一整个全部打包放进 U 盘，塞进新电脑同样的沙盒位置里，你的新软件只要一打开，连着那一千字的读书笔记，每一滴心血都会自己回到原来的右侧面板位置！
